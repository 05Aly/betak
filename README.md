# 🏢 منصة "الهضبة العقارية" | El-Hadaba Real Estate Platform

منصة إلكترونية متكاملة لربط ملاك العقارات (البائعين) بالراغبين في الشراء (المشترين) بشكل مباشر داخل منطقة **الهضبة بالمقطم، محافظة القاهرة** بهدف إلغاء دور السماسرة تماماً.

---

## 🛠️ البنية التقنية المعتمدة (Technical Stack)
* **Backend & Core:** Django (Python) - إدارة المنطق البرمجي، الصلاحيات، وحماية الصفحات.
* **Frontend:** Django HTML Templates + Tailwind CSS (تصميم مدمج وسريع).
* **Database & Storage:** Supabase (Hosted PostgreSQL & Storage) - قاعدة بيانات وملفات سحابية.
* **Realtime Chat:** Supabase JavaScript SDK (مدمج داخل الـ Templates لشات فوري لحظي).

---

## 👥 توزيع الأدوار والمسؤوليات القاطع (Strict Separation of Roles)

تم فصل المهام تماماً بين **علي** و**أحمد سمير** ليعمل كل طرف بشكل مستقل تماماً دون تداخل:

### 🛠️ أولاً: مهام علي (Ali) - [البيئة، الواجهات، الربط الجبهي، الشات، النشر]
1. **إعداد البنية التحتية السحابية:** إنشاء مشروع Supabase، إعداد قاعدة البيانات (PostgreSQL)، وتهيئة الـ Storage لرفع الملفات، وتوفير بيانات الاتصال جاهزة لأحمد.
2. **تصميم وبناء الواجهات (UI/UX Templates):** بناء كافة قوالب الـ HTML (الهوم بيج، تفاصيل العقار، صفحة الإضافة، التسجيل والدخول) وتنسيقها بـ Tailwind CSS باستخدام تاجات `x-color`.
3. **الـ Frontend Views & Context:** كتابة الـ Views المسؤولة عن رندرة الصفحات وتمرير البيانات من قاعدة البيانات إلى الـ Templates باستخدام لغة قوالب دجانغو.
4. **نظام الشات الفوري (Realtime Chat):** دمج Supabase SDK في الواجهة لتشغيل الـ Pop-up Chat اللحظي وعرض بيانات البائع.
5. **الإنتاج والنشر (Deployment):** رفع المشروع بالكامل وتجهيزه للـ Production على منصات الاستضافة (مثل Render أو Railway).

### ⚙️ ثانياً: مهام أحمد سمير (Ahmed Samir) - [بنية الـ Backend، الأمان، التحقق، لوحة الأدمن]
1. **تهيئة مشروع Django:** إنشاء المشروع برمجياً، وربطه ببيانات اتصال Supabase التي جهزها علي، وتثبيت الحزم اللازمة.
2. **بنية البيانات والأمان (Core Backend & Auth):** بناء الـ Custom User Model والـ Models الخاصة بالعقارات والمحادثات، وبناء نظام الـ Login/Register برمجياً بالكامل وتحديد أنواع الحسابات (بائع/مشتري).
3. **حماية المسارات والصلاحيات (Security Guards):** إعداد الـ Decorators (مثل `@login_required`) لمنع الزوار من رؤية تفاصيل العقارات إلا بعد تسجيل الدخول، والتحكم في الميزات (مثل إظهار ميزة الفلترة للمشتري فقط).
4. **منطق الـ Validation الصارم:** كتابة شروط الأمان الخلفية لمنع رفع أي فيديو يتجاوز حجمه **10MB**، وجعل الحسابات والإعلانات الجديدة بحالة معلقة (`False` / `waiting_list`).
5. **لوحة تحكم الإدارة (Django Admin Panel):** إعداد وتخصيص لوحة الأدمن بالكامل لتمكين أصحاب الموقع من مراجعة وقبول الحسابات الجديدة والعقارات المعلقة.

## 🚀 خطة التطوير البرمجي المنفصلة خطوة بخطوة (Step-by-Step Execution Plan)

### 📍 المرحلة 1: إعداد السحابة وقاعدة البيانات ── [مسؤولية: علي]
1. إنشاء الحساب والمشروع على منصة Supabase.
2. تفعيل الـ PostgreSQL Database والـ Storage Buckets.
3. استخراج الـ Database Connection String وإرسالها كاملة لأحمد سمير للبدء في الـ Backend.

### 📍 المرحلة 2: تهيئة الـ Backend وهيكلة البيانات ── [مسؤولية: أحمد سمير]
1. إنشاء مشروع Django وتضمين الـ Connection String الخاصة بـ Supabase في الـ `settings.py`.
2. بناء الـ Custom User Model ببياناته (الاسم رباعي، الهواتف، نوع الحساب، الصورة الإجبارية، وحقل `is_approved=False`).
3. بناء الـ Property Model بحقول السعر والعنوان والوصف وحالة الـ `waiting_list`.
4. كتابة الـ Clean functions/Validators لمنع قبول أي فيديو أكبر من **10MB** من جهة السيرفر.

### 📍 المرحلة 3: لوحة التحكم ونظام الحماية ── [مسؤولية: أحمد سمير]
1. تسجيل الـ Models في الـ `admin.py` وتخصيص الواجهة ليعرض الحسابات المعلقة والإعلانات المعلقة بشكل يسهل على الأدمن مراجعتها والموافقة عليها.
2. كتابة الـ Logic الخاص بالـ Authentication (التحقق من رقم الهاتف أو الإيميل مع الباسورد).
3. تطبيق حماية `@login_required` على الـ Views التي تتطلب حساباً لعرضها.

### 📍 المرحلة 4: تصميم وبناء القوالب والربط الجبهي ── [مسؤولية: علي]
1. تصميم الـ Homepage لعرض الإعلانات المعتمدة فقط، وإضافة رسالة إخلاء المسؤولية القانونية في الـ Footer.
2. تصميم صفحة تفاصيل العقار الكاملة، وصفحة إضافة عقار جديد.
3. كتابة الـ Views التي تقوم بجلب البيانات من الـ Models (التي صممها أحمد) وعرضها داخل الـ HTML Templates باستخدام الـ Template Tags لـ Django (مثل عمل الفلترة للمشتري بناءً على السعر `order_by`).
4. استخدام الشروط داخل الـ HTML لإخفاء زر "أضف عرضك" تماماً عن حسابات المشترين.

### 📍 المرحلة 5: تشغيل الشات الفوري اللحظي ── [مسؤولية: علي]
1. استدعاء الـ Supabase JavaScript SDK داخل قالب تفاصيل العقار عبر الـ CDN.
2. ربط زر الشات بفتح نافذة Pop-up Chat صغيرة تعمل بنظام الـ Realtime Channels لإرسال واستقبال الرسائل فوراً وبدون ريفريش.
3. جلب وصياغة بيانات المالك (الاسم رباعي، الصورة، الهاتف) لتظهر بوضوح داخل الشات للمشتري.

### 📍 المرحلة 6: الفحص النهائي والنشر للإنتاج ── [مسؤولية: علي]
1. مراجعة تجاوب كافة صفحات الـ HTML Templates مع الشاشات المختلفة (Responsive Design).
2. رفع الكود المصدري كاملاً إلى مستودع GitHub.
3. عمل الـ Deployment النهائي للموقع على السيرفر (Render/Railway) وضبط الـ Environment Variables ليكون الموقع حياً وجاهزاً للعمل في المقطم.
